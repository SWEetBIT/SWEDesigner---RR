  \subsection{Decisioni prese}
		\begin{itemize}
			\item Dopo aver valutato le funzioni del server
				e i pro e i contro fra TOMCAT e \glossaryItem{Node.js} abbiamo optato per l'utilizzo di \glossaryItem{Node.js} per il lato \glossaryItem{back-end} per via delle motivazioni descritte nella \versioneST().
			\item Abbiamo ricercato tecnologie utili per il lato \glossaryItem{front-end} dell'applicazione ed infine abbiamo optato
				per l'utilizzo di \glossaryItem{MEAN}, uno stack comprendente \glossaryItem{MongoDB}, \glossaryItem{Express.js}, \glossaryItem{Angular 4.0} e \glossaryItem{Node.js} per via delle motivazioni descritte nella \versioneST.
			\item Dopo aver analizzato le scarse  proposte \glosasryItem{open source} per quanto abbiamo scelto di
				utilizzare la libreria grafica JGraph per via delle motivazioni descritte nella \versioneST().
			\item Abbiamo analizzato diversi \glossaryItem{template} engine compatibili con \glossaryItem{Node.js} che potessero fare al caso nostro,
				quindi essere utili con la sintassi \glossaryItem{Java}.
				Abbiamo optato per l'utilizzo di Mustache come \glossaryItem{template} engine per via delle motivazioni descritte nella \versioneST().
			\item Abbiamo affrontato il problema della validità dei file importati dagli utenti dopo l'esportazione.
				Abbiamo deciso di criptare tali file per essere certi della loro validità e dopo molte discussioni in merito al metodo di criptazione
				abbiamo optato per l'utilizzo di una tabella di \glossaryItem{hash}.
		\end{itemize}
